# Reading Books For Programmers

## 개발 서적 읽기 🦁

- 매일 각자 정한 분량의 개발 서적을 읽고 정리하는 스터디입니다.
- 정리본의 양식은 `MarkDown`양식을 따릅니다.

## 프로젝트 구조 🐥

- `책이름/장. 제목`의 양식을 따릅니다.

> e.g. 이펙티브 자바 3판/02장. 객체 생성과 파괴

## 커밋 규칙 👨‍💻

- `[장] 제목 - 깃허브아이디`의 양식을 따릅니다.

> e.g. [02장] 객체 생성과 파괴 - harry-bro
